# 🔧 Устранение проблем

## Ошибка: "threads can only be started once"

Эта ошибка возникает, когда планировщик APScheduler пытается запуститься повторно.

### Решение 1: Остановить предыдущий процесс

**Windows PowerShell:**
```powershell
# Найти процесс Python с main.py
Get-Process python | Where-Object {$_.CommandLine -like "*main.py*"}

# Или найти все процессы Python
Get-Process python

# Остановить процесс по PID (замените <PID> на реальный номер)
Stop-Process -Id <PID> -Force

# Или остановить все процессы Python (осторожно!)
Get-Process python | Stop-Process -Force
```

**Windows CMD:**
```cmd
tasklist | findstr python
taskkill /PID <PID> /F
```

### Решение 2: Перезапустить терминал

1. Закройте текущий терминал
2. Откройте новый терминал
3. Активируйте виртуальное окружение (если используете)
4. Запустите бота снова

### Решение 3: Использовать другой порт/настройки

Если проблема сохраняется, возможно конфликт с другими процессами. Проверьте:

```powershell
# Проверить, какие порты заняты
netstat -ano | findstr :<PORT>
```

### Решение 4: Временное отключение планировщика

Если планировщик не критичен для тестирования, можно временно закомментировать его запуск в `main.py`:

```python
# scheduler.start()  # Временно отключено
```

## Другие частые проблемы

### Бот не отвечает на команды

1. Проверьте, что бот запущен (нет ошибок в консоли)
2. Убедитесь, что вы пишете правильному боту
3. Проверьте логи на наличие ошибок

### Ошибка подключения к API

- **Gemini API**: Проверьте квоты на [Google AI Studio](https://aistudio.google.com/app/apikey)
- **Telegram API**: Проверьте интернет-соединение и валидность токена

### Ошибки импорта модулей

```powershell
# Переустановите зависимости
pip install -r requirements.txt --force-reinstall
```

## Логи и отладка

Включите детальное логирование, изменив уровень в `main.py`:

```python
logging.basicConfig(
    level=logging.DEBUG,  # Вместо INFO
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
)
```

## Проверка настроек

Запустите скрипт проверки:

```powershell
python check_setup.py
```

Он проверит:
- ✓ Версию Python
- ✓ Наличие файла .env
- ✓ Корректность API ключей
- ✓ Установленные зависимости
- ✓ Наличие всех файлов проекта
